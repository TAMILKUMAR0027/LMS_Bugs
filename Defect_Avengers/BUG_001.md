## Summary

## Preconditions
- User is on any data-driven listing page (e.g., Course Structure, Dynamic Field Settings).
---
## Steps to Reproduce
1. Navigate to a data-driven page (e.g., Course Structure or Dynamic Field Settings).
2. Apply a search/filter value that has no matching records (e.g., a keyword or category that does not exist), or open a category/section that currently has zero records.
3. Observe the grid/list area.
---
## Expected Result
The UI should display a clear empty-state message such as 'No data found' or 'No records match your search', so the user knows the search/filter completed and simply returned nothing.
 
---
## Actual Result
The grid/list area appears blank with no message, no icon, and no indication of whether the page loaded correctly or whether there is genuinely no data.

## Impact
Without an empty-state indicator, users and testers cannot distinguish a genuinely empty result set from a broken or still-loading page, reducing usability and increasing support/debugging overhead.
